## Research method

Current official sites were opened in Chrome at 1440px and their live header/logo applications were inspected. No logo-aggregation site was used. Temporary screenshots and DOM metadata stayed outside the repository; this document records only links, observations, strict reference roles, and no-copy boundaries.

The objective is not to find a logo to imitate. It is to stop blind prompt generation and identify which identity decisions consistently work in this category.

## Three primary reference roles

1. **Booksy — category positioning:** wordmark-first service identity with one restrained custom gesture.
2. **Gusto — structural construction:** friendliness and ownership created by custom letter rhythm, not an arbitrary icon.
3. **Square — application system:** derive a tiny glyph only after the core identity works, then keep glyph/wordmark roles disciplined.

SQUIRE is the deliberate anti-reference: Barber Queue should not become macho, condensed, dark, or barber-culture performative.

## Art-direction conclusion

The failed rounds asked the wrong first question: “What abstract symbol represents a queue?” Current category evidence points to a better starting point: **the name itself should carry the identity**.

### Recommended thesis — Queue Word Rhythm

Start with a custom wordmark, not a standalone icon:

- primary lockup text: `barber queue`;
- friendly practical lowercase construction with adult weight and compact width;
- use the natural repeated rhythm inside `queue` (`u-e-u-e`) as the ownable product idea;
- introduce only one controlled “next turn” gesture—a spacing cut, terminal, or restrained apricot state change inside the word `queue`;
- no separate Q circle, magnifier, comb, chair, barber tool, or generated abstract mark;
- no raw font logo: begin from a strong source typeface, then deliberately redraw only the few letters needed to own the rhythm.

This direction connects the category, product action, and name without inventing a metaphor the user must decode.

### Derived glyph rule

Do not design a favicon/app mark independently. After the wordmark passes, derive a compact glyph from the approved custom letter system—likely the exact `bq` relationship or one queue-rhythm module. This follows Square's application discipline without borrowing Square geometry.
