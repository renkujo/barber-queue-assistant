# คู่มือทดลองใช้สำหรับเจ้าของร้าน — 1 หน้า

สถานะ: **คู่มือเตรียมทดลองเท่านั้น ยังไม่อนุญาตเริ่ม Pilot**

## เปิดร้าน

1. เข้า `/owner` และล็อกอิน
2. ตรวจเวลาเปิดร้าน บริการ และสถานะเปิดรับจอง/วอล์กอินของวันนี้
3. ถ้าหน้าคิวเปิดไม่ได้ ให้ใช้กระดาษหรือ LINE แบบเดิมทันที และอย่ากดสร้างคิวเดิมซ้ำ

## รับลูกค้า

- ลูกค้าจองหรือรับคิวเองผ่านลิงก์ได้
- ลูกค้าที่โทร แชต หรือเดินเข้าร้าน เจ้าของเพิ่มคิวให้ได้จากหน้าร้าน
- ส่งลิงก์ติดตามคิวให้ลูกค้าเมื่อเหมาะสม
- ไม่ต้องบังคับลูกค้าทุกคนให้ใช้ช่องทางเดียวกัน

## อัปเดตคิว

- `รออยู่` — อยู่ในลำดับคิว
- `มาถึงแล้ว` — ลูกค้าอยู่ที่ร้านแล้ว
- `กำลังตัด` — เริ่มให้บริการแล้ว
- `เสร็จแล้ว` — งานจบแล้ว
- `มาสาย` / `ยกเลิก` / `ไม่มา` — ใช้ตามเหตุการณ์จริง

ให้มีคิว `กำลังตัด` เพียงคนเดียว และอัปเดตทันทีที่สถานะจริงเปลี่ยน

## ถ้าระบบมีปัญหา

1. หยุดรับคิวออนไลน์ชั่วคราว
2. ใช้ fallback ที่ตกลงไว้: กระดาษหรือ LINE เดิม
3. จดเฉพาะเวลาเกิดเหตุ จำนวนคิวที่อยู่นอกระบบ และอาการ—ไม่จดข้อมูลลูกค้าเพิ่ม
4. แจ้ง Kiattisak
5. อย่าสร้างคิวเดิมซ้ำเพื่อทดลองว่าระบบกลับมาหรือยัง

## หยุดทดลองทันทีเมื่อ

- คิวหายหรือมี booking ชนกัน
- อัปเดตผิดคน
- ลูกค้าเห็นข้อมูลของคนอื่น
- กู้คิวที่กำลังใช้งานของวันนั้นไม่ได้
- ระบบทำให้ร้านสับสนกว่าวิธีเดิม

## ก่อนปิดร้าน — ไม่เกิน 3 นาที

บอก Kiattisak เพียง:

- วันนี้มีคิวจริงนอกระบบกี่คิว
- มีคิวหาย ซ้ำ ชน หรือผิดคนหรือไม่
- ลูกค้าถามซ้ำ/ขอความช่วยเหลือเรื่องคิวมากหรือไม่
- ระบบช่วยลดเวลาหรือความวุ่นวายหรือไม่
- ความมั่นใจวันนี้ `1–5` และปัญหาสำคัญที่สุดหนึ่งเรื่องโดยไม่ระบุข้อมูลลูกค้า

เป้าหมายไม่ใช่ใช้ระบบให้ครบทุกฟีเจอร์ แต่ดูว่ามันช่วยงานร้านจริงหรือไม่
