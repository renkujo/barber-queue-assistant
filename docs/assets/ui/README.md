# Pastel Paper Queue UI Assets — Draft SVG Pack

Status: draft/reference only. Kiattisak rejected this first SVG direction as not precise enough. Use `../generated-v1/` as the current preferred asset pack.


Initial source-owned SVG asset pack for Barber Queue Assistant.

These assets follow the Pastel Paper Queue direction:

- warm cream and paper surfaces;
- cocoa line art;
- peach/apricot accents;
- sage/mint positive states;
- dusty rose warning;
- no emoji-based icons.

## Files

- `logo-mark.svg` — app/brand mark
- `barber-pole.svg` — shop/status cue
- `queue-ticket.svg` — queue code/tracking cue
- `barber-chair.svg` — empty/status illustration
- `scissors-comb.svg` — service selection cue
- `line-notification.svg` — optional LINE notification cue
- `no-show-warning.svg` — late/no-show warning cue
- `walk-in-queue.svg` — walk-in/today queue cue

## Usage notes

- Use SVG directly for prototypes and early app implementation.
- Keep the original SVG files as editable source assets.
- When the app scaffold exists, copy production-ready assets into the app's `public/` or asset directory.
- If PNG variants are needed later, export at 1x/2x/3x from these SVGs or from a refined vector source.
- Do not recolor into blue/purple SaaS tones; keep the pastel system consistent.
