คู่มือตั้งแต่สร้าง OA และ channels ไปจนถึง LIFF identity, webhook, rich-menu URLs, owner alerts, NotificationLog และ real-send smoke

02

Implemented vs Planned

อย่าสับสนระหว่างสิ่งที่ใช้งานได้แล้วกับสิ่งที่ยังต้องทำด้วยมือหรือยังไม่อนุมัติ

Implemented

พร้อมใช้งานใน source

  • POST /api/line/webhook + signature verification
  • Bind follow/message user IDs
  • /line targets: book, walk-in, queue-status, owner
  • Server verification ของ LIFF ID token
  • Purpose-bound HttpOnly identity cookies
  • LINE text push + NotificationLog
  • One-use owner LINE connection from settings
Manual / not implemented

อย่าสัญญาว่ามีแล้ว

  • Rich menu creation automation — ต้องสร้างใน LINE Manager เอง
  • Production retry worker สำหรับ failed push
  • Full chatbot/chat-booking conversation
  • Payment/deposit ผ่าน LINE
  • Pilot measurement/customer execution — ต้อง approval แยก
Docs drift ที่ต้องรู้เอกสารเก่าบางจุดบอกว่า owner notification UI ยังไม่มี แต่ source ปัจจุบันมี flow เชื่อม owner LINE จาก /owner/settings แล้ว

04

ตั้งค่า LINE ทีละขั้น

ทำตามลำดับนี้เพื่อไม่สลับ Messaging API channel กับ LINE Login channel

  1. สร้าง Provider ใน LINE Developers Console เช่น Barber Queue Assistant
  2. สร้างหรือเลือก LINE Official Account ของร้านใน LINE Official Account Manager
  3. เปิด Messaging API และเชื่อมกับ Provider เดียวกัน
  4. คัดลอก Channel secret และ issue Channel access token
  5. สร้าง LINE Login channel ใน Provider เดิม เลือก App type เป็น Web app
  6. สร้าง LIFF app: Size Full, Endpoint https://<domain>/line, scopes openid + profile, Add friend On (Normal)
  7. ใส่ env ทั้งสามค่าแล้ว restart/redeploy
  8. ตั้ง webhook URL และเปิด Use webhook
  9. กด Verify ใน LINE Developers และตรวจ server response
  10. สร้าง rich menu ด้วย URLs ของ target ที่ระบบรองรับ
  11. Login owner แล้วเชื่อม LINE เจ้าของร้านจาก /owner/settings
  12. ทำ real-send smoke ด้วย test LINE account ก่อนเปิดให้ร้านใช้

เชื่อม LINE เจ้าของร้าน

ใช้ UI flow เป็นทางหลัก OWNER_LINE_USER_ID เป็น fallback เท่านั้น

/owner/settingsOwner clicks connect
One-use tokenRandom nonce · 10 min
LIFF owner targetVerified LINE ID
Atomic consumeReplay blocked
ShopSettingsownerLineUserId saved
  1. Owner login แล้วเปิด /owner/settings
  2. กด เชื่อม LINE เจ้าของร้าน
  3. ระบบออก signed token ที่มี random nonce และ expiry 10 นาที
  4. เปิด LIFF ใน LINE account ที่ต้องการรับแจ้งเตือน
  5. Completion route consume nonce และ update ShopSettings.ownerLineUserId ใน transaction เดียว
  6. หน้า result แสดง connected/missing-line/invalid ด้วย signed result token อายุ 2 นาที
Owner alert scopeแจ้ง owner เฉพาะ booking/walk-in ที่ลูกค้าสร้างเอง Owner-created walk-in ไม่ push กลับหา owner เพื่อไม่สร้าง self-notification spam
07

Identity และ Security

ระบบไม่เชื่อ lineUserId จาก URL หรือ hidden input

  • LIFF ส่ง ID token เข้า server action; server เรียก https://api.line.me/oauth2/v2.1/verify
  • ตรวจ aud เท่ากับ LINE Login channel ID, iss, exp และ sub
  • Identity ถูกเก็บใน signed HttpOnly cookie แยก purpose: book, walk-in, owner
  • Cookie อายุ 10 นาที, SameSite=Lax และ Secure ใน production
  • Booking/walk-in ใช้ verified cookie แล้ว clear หลังสำเร็จ; recoverable retry ยังรักษา identity
  • QueueItem snapshot lineUserIdSnapshot เพื่อไม่พึ่ง customer row ที่เปลี่ยนได้
  • Owner connection token ต้อง signature ถูก, ยังไม่หมดอายุ, nonce มีอยู่และยังไม่ consumed
  • Webhook ตรวจ HMAC SHA-256 แบบ timing-safe เมื่อ channel secret ถูกตั้ง
  • ห้าม log access token, channel secret, ID token, signed cookie หรือ raw LINE user IDs โดยไม่จำเป็น

NotificationLogความหมาย
NONE / SKIPPEDไม่มี customer LINE identity
LINE / SKIPPEDมี recipient แต่ไม่มี access token
LINE / SENTLINE API accepted request — ไม่ใช่หลักฐานว่า user อ่านหรือได้รับแน่นอน
LINE / FAILEDProvider call ถูก reject/timeout/error
LINE / PENDINGPilot at-most-once attempt outcome ยังไม่ทราบ; ห้าม resend อัตโนมัติ
PIN scopeส่ง PIN จริงเฉพาะข้อความเริ่มต้นไป verified customer LINE เท่านั้น ข้อความสถานะภายหลังไม่ส่งซ้ำ และ NotificationLog เก็บเพียง PIN เช็คคิว: ••••
Fail-soft contractLINE failure ต้องไม่ block booking, walk-in หรือ owner status action ให้ตรวจจาก NotificationLog ภายหลัง

Troubleshooting

ใช้ NotificationLog และ response status แยกปัญหา identity, credentials และ provider

Webhook Verify ไม่ผ่าน
ตรวจ URL ลงท้าย /api/line/webhook, HTTPS domain/tunnel, server reachability และ Channel secret ว่ามาจาก Messaging API channel เดียวกัน
LIFF ไม่ redirect หรือขึ้น fallback
ตรวจ LIFF ID, endpoint URL, scopes openid/profile, เปิดจาก LINE app และ rebuild หลังแก้ NEXT_PUBLIC_LINE_LIFF_ID
LINE webview ใช้ dev mode แล้ว chunk/error
ทดสอบด้วย production local mode: pnpm build แล้ว pnpm start จากนั้นเปิดผ่าน HTTPS tunnel
Notification ไม่เข้า
NONE/SKIPPED = ไม่มี identity, LINE/SKIPPED = token หาย, LINE/FAILED = provider reject, LINE/SENT = app ส่งถึง LINE API แล้ว ให้ตรวจ account ปลายทางต่อ
Owner connect ขึ้น missing-line หรือ invalid
missing-line มักเกิดจากไม่ได้เปิดใน LINE; invalid มักเป็น token หาย/หมดอายุ/replay/tamper ให้กลับไปสร้างลิงก์ใหม่จาก owner settings
Owner ไม่ได้รับ alert
ตรวจ ShopSettings.ownerLineUserId หรือ fallback, access token และยืนยันว่า event มาจาก customer-created booking/walk-in ไม่ใช่ owner-created queue

Production checklist

รายการสั้นสำหรับ operator ก่อนเปิด rich menu หรือ webhook ให้ลูกค้าจริง

  • Public HTTPS domain และ /api/health ทำงาน
  • Messaging API secret/token ตั้งใน production secret store ไม่ใช่ git
  • LINE Login channel และ LIFF endpoint ใช้ domain ปัจจุบัน
  • NEXT_PUBLIC_LINE_LIFF_ID ถูก rebuild เข้า client bundle
  • Webhook secret ตั้งแล้วก่อน enable Use webhook
  • Webhook Verify ผ่านและ invalid signature ตอบ 401
  • Rich menu links ใช้ /line?target=walk-in และ /line?target=book
  • Customer real-send และ owner connection smoke ผ่าน
  • NotificationLog status ถูกต้องทั้ง SENT/SKIPPED/FAILED
  • Normal web booking/walk-in/tracking ยังทำงานเมื่อ LINE unavailable
  • Privacy contact เป็นช่องทางจริง และ Pilot Measurement ยัง disabled
