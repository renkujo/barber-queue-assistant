01

ภาพรวมและขอบเขต

ระบบคิวสำหรับร้านตัดผมเจ้าของคนเดียว ไม่ใช่ salon ERP และไม่บังคับให้ลูกค้าทุกคนใช้ LINE

Customer

จองและรับบัตรคิว

ลูกค้าเช็คสถานะร้าน จองล่วงหน้า รับคิววันนี้ และติดตามด้วย public token หรือ queue code + PIN

Owner

จัดการคิววันนี้

Owner login ด้วย passcode, เพิ่ม walk-in, จัดลำดับ, เริ่มงาน, ปิดงาน, late, no-show และตั้งค่าร้าน

Channel

เว็บเป็นแกนหลัก

LINE OA/LIFF เป็นช่องทางเข้าและแจ้งเตือน ระบบเว็บและ tracking code ต้องยังใช้ได้เมื่อ LINE ไม่พร้อม

Operational boundaryPilot Measurement, Pilot Execution, invitation, Daily Close external collection และ production promotion ต้องมี approval แยกจากการ deploy ตัวแอป
02

เริ่มต้นพัฒนาใน local

ทุกคำสั่ง frontend/app รันจากโฟลเดอร์ app/ และใช้ pnpm ตาม lockfile

Runtime

Node.js 22+ และ Corepack/pnpm 10.33.0

Services

PostgreSQL ที่เข้าถึงได้ผ่าน DATABASE_URL

Install และ run

cd app
pnpm install --frozen-lockfile
cp .env.example .env

# เติมเฉพาะค่า local ของคุณ ห้าม commit
pnpm prisma:generate
pnpm db:migrate
pnpm db:seed
pnpm dev

เปิด http://localhost:3000 และ owner login ที่ /owner/login

ห้ามเผยแพร่ secretใช้ app/.env.example และ .env.production.example เป็นรายการชื่อเท่านั้น ค่าใน .env จริงต้องไม่เข้า git, screenshot, log หรือเอกสาร

08

Security invariants

สิ่งที่ห้ามทำให้หลุดระหว่าง refactor

  • Owner mutations เรียก requireOwnerSession() ก่อนแตะข้อมูล
  • Production session cookie ใช้ HTTPS/secure และ secret แยกจาก passcode
  • Public tracking รับ UUID public token ไม่ใช่ database ID และมี noindex
  • Queue lookup ใช้ queue code + 4-digit PIN และ rate limit ต่อ IP/code
  • Raw IP ไม่ถูกเก็บใน RateLimitBucket; ใช้ hashed fingerprint
  • LINE identity มาจาก server-verified ID token ไม่รับ query/hidden lineUserId
  • Notification failure ต้องไม่ rollback booking/queue mutation
  • อย่า expose container port 3000 ตรงสู่ internet; trusted proxy เป็น client-IP boundary
  • อย่า commit secret, production dump, browser auth state หรือ generated test artifacts

Operations checklist

ขั้นต่ำก่อนส่งต่อระบบให้สภาพแวดล้อมจริง

  • /api/health ตอบ success, database reachable และ pilot disabled
  • Owner login, customer walk-in, owner start/close และ tracking link ทำงานครบ
  • /api/queue/status ไม่เผย customer/queue detail fields
  • /privacy แสดงช่องทางติดต่อจริง
  • LINE ว่างได้โดย queue action ไม่พัง; NotificationLog บันทึก skipped
  • R2 backup คง disabled จน config + backup + checksum + lifecycle ผ่าน
  • ตั้ง uptime monitor สำหรับ HTTP 503 ของ /api/health
  • หยุด dev server/Playwright และตรวจ port 3000 หลัง validation

Troubleshooting

เริ่มจาก boundary ที่เล็กที่สุดและอย่ากลบ error ด้วย fallback

Prisma ใช้งานไม่ได้หรือ app start ไม่ขึ้น
ตรวจ DATABASE_URL, PostgreSQL health, migration logs และ pnpm prisma:validate ก่อน ถ้า production start ค้างให้ดู prisma migrate deploy
Owner login วนกลับหน้า login
ตรวจ passcode/session secret, HTTPS และ secure-cookie behavior ห้ามลด security flag เพื่อแก้ production โดยไม่เข้าใจ proxy
หน้า customer ใช้ mock/fallback data
Repository บางจุดมี safe fallback เมื่อ DB unavailable ตรวจ database connection/log ก่อนเชื่อข้อมูลในหน้า
Responsive ผ่าน overflow แต่ card ยังดูชนกัน
ตรวจ screenshot จริงที่ 360/390/768/1024/1440, sibling gaps, Thai wrapping, fixed-nav clearance และ card height growth
LINE ไม่ส่งแต่ queue ถูกสร้าง
เป็น fail-soft contract ที่ตั้งใจ ตรวจ NotificationLog, recipient snapshot และ token ต่อใน LINE OA Guide
