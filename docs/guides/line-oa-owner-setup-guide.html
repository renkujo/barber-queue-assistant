Owner + Developer ข้อตกลง ownership ใครเป็นเจ้าของอะไร เตรียมก่อนติดตั้ง Owner สร้างบัญชี เชิญ Developer Developer ตั้งค่า ติดตั้งเข้าแอป ทดสอบร่วมกัน ส่งมอบและถอนสิทธิ์ Troubleshooting

Owner owns · Developer configures

เจ้าของร้านถือบัญชี เราติดตั้งระบบให้ โดยไม่ต้องขอรหัสผ่าน

คู่มือการสร้าง LINE OA และ LINE Developers แบบที่เจ้าของร้านควบคุมบัญชี การกู้คืน และสิทธิ์ระยะยาว ส่วน Developer รับสิทธิ์ชั่วคราวเพื่อเชื่อม Messaging API, LIFF, webhook และระบบแจ้งเตือน

01

ข้อตกลง Ownership

ตกลงเรื่องเจ้าของบัญชีก่อนเริ่มสร้าง Provider หรือ Channel

Recommended

Owner owns

  • LINE OA อยู่ใต้บัญชีที่เจ้าของร้านควบคุม
  • Provider/Channels สร้างใน business identity ของร้าน
  • เจ้าของร้านเป็น Admin หลักและมี recovery method
  • Developer ได้รับสิทธิ์ผ่าน invitation
Avoid

Developer owns

  • อย่าสร้าง OA/Provider ใต้ LINE ส่วนตัวของ Developer
  • อย่าใช้บัญชีร่วมที่ทุกคนรู้ password เดียวกัน
  • อย่าฝาก recovery email/2FA ไว้กับผู้รับจ้างคนเดียว
  • อย่าพึ่ง workflow การโอน channel ภายหลัง
Developer ทำแทนได้ แต่ต้องอยู่ใน ownership ของร้านถ้าเจ้าของร้านไม่ถนัด ให้ทำผ่าน screen share หรือให้ Owner login ด้วยตัวเองแล้วเชิญ Developer เข้าระบบ ห้ามขอให้ส่ง password/OTP ในแชต

03

เตรียมก่อนวันติดตั้ง

เตรียมข้อมูลให้ครบเพื่อจบ setup ในหนึ่ง session โดยไม่ส่ง credential ผ่านแชต

เจ้าของร้านเตรียม

  • LINE account หรือ LINE Business ID ที่จะถือ OA
  • Email/เบอร์ recovery ที่ร้านเข้าถึงได้
  • เปิด 2FA ตามที่ platform รองรับ
  • ชื่อร้าน รูปโปรไฟล์ และข้อมูลติดต่อ
  • มือถือที่ติดตั้ง LINE สำหรับทดสอบ LIFF
  • ตกลงว่า LINE account ใดรับ owner alerts

Developer เตรียม

  • Public HTTPS domain หรือ HTTPS tunnel
  • สิทธิ์ deploy/Dokploy secret store
  • App รันและ database health ผ่าน
  • Webhook และ LIFF URLs ที่จะใช้
  • Test customer LINE account แยกจาก owner ถ้าเป็นไปได้
  • Smoke checklist และ rollback plan
ห้ามเตรียมด้วยการส่ง passwordOwner เป็นคนกรอก password/OTP เอง Developer ใช้ role invitation หรือ screen share เท่านั้น
04

Owner สร้างบัญชีและ OA

ขั้นตอนนี้ควรทำโดยเจ้าของร้าน หรือทำพร้อม Developer ผ่าน screen share

  1. เข้า LINE Official Account Manager ด้วยบัญชีที่ร้านควบคุม
  2. สร้าง LINE Official Account ชื่อร้าน เลือกประเทศและหมวดธุรกิจที่เหมาะสม
  3. ยืนยันว่า Owner เปิดหน้า OA Manager ได้และ recovery method เป็นของร้าน
  4. เข้า LINE Developers Console
  5. สร้าง Provider เช่น ชื่อร้าน หรือ Barber Queue Assistant — ชื่อร้าน
  6. จดเฉพาะชื่อ Provider/OA ใน checklist ไม่จด token หรือ secret ลงเอกสาร
ถ้ามี OA เดิมอยู่แล้วใช้ OA เดิมของร้านได้ ไม่ต้องสร้างซ้ำ ให้ตรวจว่าเจ้าของร้านมี Admin/recovery ก่อนเปิด Messaging API
05

Owner เชิญ Developer

เมนู/ชื่อ role อาจต่างตามภาษาและ UI เวอร์ชัน ให้ยึดหลัก least privilege

LINE Official Account Manager

  1. Owner เปิด OA → Settings → Manage permissions/จัดการสิทธิ์
  2. สร้าง invitation สำหรับ Developer ตามวิธีที่หน้า Manager รองรับ
  3. ใช้สิทธิ์สูงเท่าที่จำเป็นชั่วคราวสำหรับ Messaging API/rich menu
  4. Owner ยังคงเป็น Admin หลักตลอดเวลา

LINE Developers Console

  1. Owner เปิด Provider → Roles/สมาชิก
  2. เชิญ Developer ด้วย business identity/email ที่ระบุตัวได้
  3. ถ้าต้องสร้าง Channel ให้มอบสิทธิ์ที่ทำได้ชั่วคราว แล้วลดสิทธิ์หลังส่งมอบ
  4. Developer ทดสอบว่าเห็น Provider แต่ไม่ต้องรู้ password ของ Owner
บันทึกสิทธิ์อย่างง่ายจดว่าใครมีสิทธิ์อะไร วันที่ให้สิทธิ์ และวันที่จะ review/ถอนสิทธิ์ แต่ไม่จด credential
06

Developer ตั้งค่า Channels

สร้าง Messaging API และ LINE Login/LIFF ภายใต้ Provider ของร้าน

Messaging API

Webhook + Push

  1. Enable Messaging API จาก OA
  2. ตรวจว่า channel อยู่ Provider ของร้าน
  3. คัดลอก Channel secret เข้า secret store
  4. Issue access token เข้า secret store
  5. ตั้ง webhook URL หลัง public HTTPS พร้อม
LINE Login

LIFF Identity

  1. สร้าง LINE Login channel ใน Provider เดิม
  2. App type: Web app
  3. สร้าง LIFF app ขนาด Full
  4. Endpoint: https://<domain>/line
  5. Scopes: openid + profile

URLs ที่ต้องตั้ง

LIFF endpoint
https://<domain>/line

Webhook URL
https://<domain>/api/line/webhook

Rich menu
https://<domain>/line?target=walk-in
https://<domain>/line?target=book
https://<domain>/line?target=queue-status
อย่า enable production webhook ก่อน secret พร้อมSource ปัจจุบันตรวจ signature เมื่อ LINE_CHANNEL_SECRET ถูกตั้งเท่านั้น
07

ติดตั้งค่าเข้าแอป

Developer ใส่ค่าลง local env หรือ Dokploy secret store โดยไม่ส่งค่าจริงกลับมาในเอกสาร

LINE_CHANNEL_SECRET="replace-with-channel-secret"
LINE_CHANNEL_ACCESS_TOKEN="replace-with-access-token"
NEXT_PUBLIC_LINE_LIFF_ID="1234567890-example"

# Optional fallback only
OWNER_LINE_USER_ID=""
  1. Local: ใส่ค่าใน app/.env ที่ไม่ถูก commit
  2. Production: ใส่ค่าใน Dokploy environment/secret store
  3. Restart หรือ rebuild/redeploy หลังแก้ NEXT_PUBLIC_LINE_LIFF_ID
  4. เปิด /api/health และหน้าเว็บปกติก่อนตั้ง LINE URLs
  5. ตั้ง webhook และ rich menu หลัง domain version ปัจจุบันขึ้นจริง
ไม่มี token ใน git/chatถ้าต้องประสานงาน ให้ส่งเพียงสถานะ “ตั้งแล้ว/ยังไม่ตั้ง” หรือใช้ password manager/secret store ที่ได้รับอนุมัติ

09

ส่งมอบและถอนสิทธิ์

ปิดงานโดยให้ Owner พิสูจน์ว่าควบคุมระบบได้ และลด developer access ให้เหมาะสม

  • Owner login OA Manager และ LINE Developers Console ได้ด้วยตัวเอง
  • Owner เห็น Provider, Messaging API channel, LINE Login channel และ LIFF app
  • Owner มี recovery email/phone และ 2FA ที่ควบคุมเอง
  • Developer ส่งมอบรายการ URLs และชื่อ env เท่านั้น ไม่ส่ง secret ในเอกสาร
  • Owner รู้วิธีเชื่อม LINE แจ้งเตือนใหม่จาก /owner/settings
  • Owner รู้วิธีปิด rich menu/webhook ชั่วคราวเมื่อเกิด incident
  • Review role ของ Developer: ลดสิทธิ์หรือถอนออกตามข้อตกลง support
  • Rotate token/secret ถ้ามีเหตุสงสัยว่าถูกเปิดเผย
  • บันทึกวันส่งมอบ ผู้รับผิดชอบ support และช่องทาง incident

ถ้ามี support ต่อ

คง developer role ระดับต่ำสุดที่แก้ระบบได้ กำหนดวัน review และห้ามใช้ shared password

ถ้าจบงานแล้ว

ถอน developer role หลัง Owner ยืนยัน access และเก็บวิธีติดต่อกรณี incident แยกจาก platform credentials

10

Troubleshooting การส่งมอบ

ปัญหาบัญชีและสิทธิ์ต้องแก้ที่ ownership boundary ก่อน debug code

Owner สร้าง OA ไม่ได้หรือไม่เห็น Provider
ตรวจว่ากำลังใช้ business identity/LINE account ที่ถูกต้องและเลือก Provider เดียวกัน ห้ามแก้ด้วยการสร้าง Provider ใหม่ใต้บัญชี Developer โดยไม่ตกลง
Developer ไม่เห็น Channel
ให้ Owner ตรวจ role ที่ Provider/Channel และ invitation status ไม่ส่ง password เพื่อให้ Developer login แทน
อดีต Developer เป็น Admin คนเดียว
ให้เพิ่ม Owner-controlled Admin และพิสูจน์ recovery ก่อนลด/ถอนอดีต Developer ถ้าเข้าบัญชีไม่ได้ให้ใช้ official LINE support/recovery process
ต้องเปลี่ยน Developer
Owner เชิญคนใหม่ผ่าน role, ส่งมอบ URLs/architecture, rotate credentials ตามความเสี่ยง แล้วถอนคนเดิมหลัง smoke ผ่าน
Token อาจหลุด
Reissue access token, เปลี่ยน channel/session secrets ตามขอบเขตที่กระทบ, redeploy, ตรวจ NotificationLog และอย่าแปะ token ใน issue/chat
